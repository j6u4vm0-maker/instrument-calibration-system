# 檢具類別獨立化實作拆分清單

## 1. 目標
依據 [fixture-category-separation-spec.md](./fixture-category-separation-spec.md) 的規格，將檢具與設備的類別系統完全拆分。

## 2. 開發原則
- 先更新規格，再做實作
- 單一檔案避免超過 300-500 行
- 單一函式避免超過 30-50 行
- 新增 UI 必須支援中英文
- CRUD 功能預設要補齊新增、編輯、刪除

## 3. 任務拆分

### 3.1 資料層
- 新增 `FixtureCategory` 資料模型
- 補上 `Fixture.categoryId` 關聯
- 規劃設備與檢具的類別資料分離策略
- 規劃舊資料遷移腳本

### 3.2 服務層
- 拆分設備類別服務與檢具類別服務
- 每個服務都只處理自己的類別資料
- 補齊建立、更新、刪除、查詢
- 增加重複名稱與空值防護

### 3.3 Action 層
- 建立設備類別 action
- 建立檢具類別 action
- 重新整理各自模組對應的頁面
- 保持錯誤回傳格式一致

### 3.4 前端頁面
- 設備類別管理頁只讀設備類別
- 檢具類別管理頁只讀檢具類別
- 檢具新增 / 編輯頁只顯示檢具類別
- 設備新增 / 編輯頁只顯示設備類別

### 3.5 匯入 / 匯出
- 設備匯入只解析設備類別
- 檢具匯入只解析檢具類別
- 匯出檔案欄位要清楚標示所屬模組

### 3.6 驗證與測試
- 類別重複建立不應報 Prisma unique error
- 檢具頁不應顯示設備類別
- 設備頁不應顯示檢具類別
- 匯入後類別數量應正確
- 刪除後可依規格恢復或軟刪除

## 4. 推薦順序
1. 先完成資料模型拆分
2. 再拆 service 與 action
3. 再改前端類別來源
4. 再補匯入 / 匯出
5. 最後做遷移與驗證

## 5. 驗收重點
- 檢具類別清單只會出現檢具類別
- 設備類別清單只會出現設備類別
- 檢具與設備互不影響
- 所有新增、編輯、刪除路徑都已覆蓋
- 介面文字符合雙語規則
